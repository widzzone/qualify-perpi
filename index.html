Profil Aplikasi Qualify

Qualify adalah sebuah aplikasi yang disediakan oleh PERPI untuk membantu mengidentifikasi “profesional respondent” dalam event FGD atau CLT. Profesional respondent adalah responden yang dengan sengaja dan menjadikan aktifitasnya menjadi responden FGD atau CLT sebagai profesi dimana yang bersangkutan dapat mengikuti beberapa event FGD atau CLT di berbagai lokasi atau waktu yang diadakan oleh perusahaan market riset. Keberadaan respondent proffesional ini mengurangi kualitas dari FGD/CLT yang dilakukan dan sulit untuk dideteksi di masing-masing perusahaan riset.

Implementasi penggunaan aplikasi Qualify dimulai pada tahun 2015, dan tercatat penggunaannya terus meningkat dari tahun ke tahun. Di tahun 2023 PERPI bekerjasama dengan Starcore sebagai technology partner dan melakukan penyempurnaan aplikasi Qualify dengan beberapa fitur baru, salah satunya adalah fitur multiple submission. Penggunaan aplikasi akhirnya meningkat hingga 2x lipat, tercatat di akhir tahun 2025 lalu Qualify telah digunakan untuk mengidentifikasi lebih dari 200 ribu responden dalam 10 tahun terakhir.

Jenis pengguna (user roles) di aplikasi ini terbagi atas 4 kategori yaitu :

  • 1. PERPI Admin; yang hanya dapat mengakses dashaboard dan menambahkan user roles QC Manager dan PERPI Officer (Pengurus)

  • 2. PERPI Officer; yaitu pengurus PERPI yang hanya dapat mengakses dashboard saja.

  • 3. QC Manager; yaitu perwakilan utama dari member PERPI yang dapat mengelola/membuat user untuk tim nya (QC Officer), role ini juga dapat menginput responden, approval responden dan melihat semua responden yang sudah diinput oleh tim dari perusahaan yang bersangkutan saja.

  • 4. QC Officer; yaitu tim dari member PERPI yang ditugaskan oleh QC Manager (PIC Member Qualify) untuk menginput responden untuk diidentifikasi apakah duplikat (pernah didaftarkan). QC Officer juga bisa mengakses daftar semua responden yang sudah diinput oleh yang bersangkutan saja.

1. Panduan Ringkas Penggunaan Aplikasi QUALIFY

Aplikasi QUALIFY dirancang untuk membantu tim Quality Control (QC) dalam mendeteksi "Profesional Respondents". Responden profesional adalah responden yang mengikuti beberapa acara riset (seperti FGD/CLT) di tempat atau waktu yang berbeda secara berulang. Sistem ini menggunakan NIK (Nomor Induk Kependudukan) pada KTP sebagai identifier unik untuk mengecek riwayat duplikasi responden di dalam database QUALIFY.

Tiga Langkah Utama Pendeteksian

1
Submit Single/Multiple Respondents

Mengunggah data responden secara manual satu per satu atau massal menggunakan template.

2
Automatic Identification

Sistem secara otomatis akan mengidentifikasi duplikasi ID (NIK) dari database QUALIFY.

3
Approve / Reject

Menyetujui atau menolak responden yang telah diunggah berdasarkan hasil pengecekan duplikasi.

Panduan Penggunaan Sistem (Contoh: Akses QC Manager)

Berikut adalah langkah-langkah teknis penggunaan aplikasi web QUALIFY setelah Anda melakukan login:

1. Dashboard

Halaman utama ini menampilkan ringkasan data statistik aktivitas pengguna dan tren data. Informasi yang ditampilkan meliputi jumlah pengguna terdaftar, pengguna aktif (hari ini, bulan ini, tahun ini), grafik tren submit/approval, dan persentase tingkat duplikasi (Duplication Rate).

2. User Management

Menu ini digunakan untuk mengelola pengguna. Sebagai QC Manager, Anda dapat melihat daftar pengguna di perusahaan Anda dan memiliki akses untuk membuat atau mengedit akun khusus untuk QC Officer di bawah tim Anda.

3. Respondent Approval

Menu ini adalah area kerja utama untuk mengunggah dan memverifikasi data responden.

Cara Mengunggah Data Responden:

  • Klik tombol Input Form untuk memasukkan data responden baru secara manual satu per satu.
  • Klik tombol Download Template untuk mengunduh format Excel.
  • Isi data pada file Excel sesuai kolom yang disediakan.
  • Klik tombol Upload New Respondent dan unggah file Excel yang sudah terisi.

Cara Verifikasi Data:

  • Perhatikan kolom Duplicate NIK. Jika sistem menemukan riwayat NIK yang sama di database, peringatan duplikasi akan muncul.
  • Klik ikon silang (X) warna merah untuk Reject responden jika terbukti duplikat.
  • Klik ikon centang warna hijau untuk Approve responden.
4. Respondent List

Menu ini berfungsi sebagai rekapitulasi data responden. Anda dapat melihat seluruh daftar responden yang sebelumnya sudah di-approve maupun di-reject pada menu Respondent Approval.

Berikut ini panduan cara input single (satu) responden di Qualify, yang dapat melakukan proses ini hanya untuk roles QC Officer dan QC Manager saja.

Input Single Respondent

Panduan Cara Input Form Respondent

Panduan ini menjelaskan langkah-langkah untuk menginput data responden secara manual (satu per satu) ke dalam aplikasi QUALIFY guna mengecek status duplikasinya.

1. Cara Menginput Data Responden

  • Setelah login ke aplikasi, pilih menu Respondent Approval pada panel navigasi di sebelah kiri.
  • Klik tombol Input Form yang terdapat di bagian atas tabel.
  • Pada kotak dialog Input Respondent yang muncul, lengkapi formulir pendaftaran dengan informasi berikut:
    • NIK: Masukkan Nomor Induk Kependudukan responden sesuai KTP.
    • Name: Masukkan nama lengkap responden.
    • Event Date: Pilih tanggal pelaksanaan event/riset pada kalender yang disediakan.
    • City: Pilih kota pelaksanaan dari menu dropdown.
    • Project: Ketikkan nama project riset yang sedang berjalan.
    • Industry: Pilih kategori industri yang relevan dengan project tersebut dari menu dropdown.
  • Setelah semua data dipastikan terisi, klik tombol Submit.

2. Proses Verifikasi dan Persetujuan (Approve/Reject)

Setelah Anda menekan tombol Submit, sistem QUALIFY akan secara otomatis melakukan identifikasi dan langsung menampilkan kotak dialog hasil pengecekan duplikasi.

Skenario 1: Responden Unik (Tidak Terdeteksi Duplikat)

  • Jika NIK responden belum pernah terdaftar di database sebelumnya, sistem akan menampilkan pesan peringatan: "This respondent is unique".
  • Anda dapat langsung menyelesaikan proses dengan menekan tombol Approve (hijau) untuk menyetujui responden masuk ke dalam sistem.

Skenario 2: Responden Terdeteksi Duplikat

  • Jika NIK responden sudah pernah digunakan pada project atau perusahaan lain, sistem akan menampilkan pesan: "Found 1 duplicate respondent(s) in this row" beserta detail histori partisipasi responden tersebut.
  • Berdasarkan temuan histori tersebut, Anda diberikan kebebasan untuk memutuskan apakah akan menekan tombol Reject (merah) untuk menolak responden tersebut, atau tetap menekan Approve jika dirasa masih memenuhi kriteria perusahaan.

Panduan Cara Submit Multiple Responden

Panduan ini menjelaskan cara mengunggah data responden secara massal menggunakan template Excel. Cara ini lebih efisien jika Anda memiliki puluhan atau ratusan data responden yang harus diperiksa sekaligus.

Langkah 1: Unduh Template Excel

  1. Masuk ke aplikasi QUALIFY dan buka menu Respondent Approval.
  2. Klik tombol Download Template di bagian atas tabel. Anda hanya perlu mengunduh template ini satu kali.

Langkah 2: Mengisi Data pada Template Excel

Buka file Excel yang telah diunduh dan isi data responden pada kolom yang disediakan. Sistem telah dilengkapi dengan proteksi data (data validation), jadi pastikan Anda mematuhi aturan berikut agar file dapat diunggah:

  • Kolom NIK: Wajib diisi dengan 16 digit angka. Angka pertama tidak boleh 0 atau 4. Angka terakhir tidak boleh 0. Tidak boleh ada NIK ganda di dalam satu file yang sama.
  • Kolom Name: Wajib diisi minimal 3 huruf dan tidak boleh mengandung angka.
  • Kolom Event Date: Masukkan tanggal riset/FGD dalam format Date. Rentang waktu yang diizinkan adalah maksimal 7 hari ke belakang (H-7) atau maksimal 30 hari ke depan (H+30).
  • Kolom City: Harus dipilih dari daftar kota yang sudah ada pada kolom dropdown.
  • Kolom Project: Bebas diisi dengan teks apa saja.
  • Kolom Industry: Harus dipilih dari daftar industri yang tersedia pada dropdown.

Penting: Anda bisa mengecek referensi daftar kota dan industri yang sah pada sheet atau tab "Version" di dalam file Excel tersebut. Jangan mengubah nama file Excel tersebut.

Langkah 3: Mengunggah Data ke Sistem

  1. Kembali ke aplikasi QUALIFY pada menu Respondent Approval.
  2. Klik tombol Upload New Respondent.
  3. Pada kotak dialog yang muncul, klik Choose File dan pilih file Excel yang telah Anda isi tadi.
  4. Klik Submit. Jika data sesuai dengan aturan sistem, akan muncul pesan "Data imported successfully".

Panduan Cara Menangani Error Saat Upload File

Panduan ini menjelaskan langkah-langkah untuk mengatasi kendala atau error yang mungkin terjadi saat Anda mengunggah (upload) file Excel berisi data banyak responden (Multiple Respondent) ke dalam sistem QUALIFY. Terdapat dua jenis error utama yang sering terjadi beserta solusinya:

1. Error karena Nama File atau Format Tidak Sesuai (Invalid File Name)

  • Penyebab: Error ini muncul jika nama file yang Anda unggah tidak sama persis dengan file template yang diunduh dari sistem, atau jika format dan nama kolom di dalamnya telah diubah.
  • Pesan Error: Muncul kotak silang merah dengan tulisan "Validation failed. Invalid file name, please upload file RespondenList_v[versi].xlsx".
  • Solusi:
    • Pastikan nama file yang diunggah sama persis dengan template aslinya (contoh: RespondenList_v20230708.xlsx).
    • Gunakan file template asli yang diunduh dari sistem tanpa merubah struktur formatnya (sheet, baris kolom, dll.).
    • Jika Anda menyalin data responden dari file Excel lain, sangat disarankan untuk menempelkan data menggunakan metode Copy -> Paste Value agar tidak merusak format bawaan template.

2. Error karena Kesalahan Pengisian Data (Validation Failed)

  • Penyebab: Error ini terjadi jika sistem mendeteksi adanya data yang diisi tidak sesuai dengan aturan validasi pengisian (misalnya: NIK kurang dari 16 digit, nama mengandung angka, kolom kota dibiarkan kosong, atau nama industri tidak sesuai daftar).
  • Pesan Error: Muncul peringatan "Validation failed, please check file Respondent_Validasi.xlsx that was just downloaded by the system". Seluruh isi file tersebut otomatis ditolak oleh sistem.
  • Solusi:
    • Sistem akan secara otomatis mengunduh file Excel baru bernama Respondent_Validasi_[tanggal].xlsx ke perangkat Anda.
    • Buka file hasil unduhan otomatis tersebut.
    • Geser layar Excel ke sebelah kanan untuk menemukan kolom-kolom baru berisi keterangan error (seperti name_err, nik_err, city_err, dll.).
    • Kolom error ini akan mendiagnosis dan memberi tahu Anda secara spesifik letak kesalahan pada setiap baris data (contoh pesannya: "Tidak boleh berisi angka" pada nama, atau "Data city kosong").
    • Tutup file validasi tersebut, lalu buka kembali file template Excel asli Anda.
    • Perbaiki bagian data yang keliru pada file asli sesuai dengan petunjuk error yang ditemukan sebelumnya.
    • Simpan (Save) pembaruan tersebut, lalu kembali ke aplikasi QUALIFY dan ulangi proses unggah melalui tombol Upload New Respondent.

Create New QC Manager

Panduan Menambah QC Manager Baru

Panduan ini menjelaskan langkah-langkah untuk membuat dan mengelola akun pengguna dengan peran QC Manager. Hak akses untuk melakukan proses pendaftaran akun ini hanya dimiliki oleh PERPI Admin.

1. Cara Membuat Akun QC Manager Baru

  • Silakan login ke aplikasi QUALIFY menggunakan akun perpi_admin.
  • Pada panel navigasi di sebelah kiri layar, klik menu User Management. Halaman ini akan menampilkan daftar seluruh pengguna yang sudah terdaftar di sistem.
  • Untuk menambahkan pengguna baru, klik tombol Create User berwarna hijau yang berada di pojok kanan atas halaman.
  • Pada kotak dialog Create User yang muncul, lengkapi formulir berikut:
    • Name: Masukkan nama lengkap pengguna.
    • Username: Masukkan username yang akan digunakan pengguna untuk login.
    • Email: Masukkan alamat email aktif milik pengguna.
    • Password: Buat kata sandi untuk pengguna tersebut (minimal 7 karakter).
    • Confirm Password: Ketik ulang kata sandi yang sama untuk konfirmasi.
    • Role: Klik menu dropdown dan pilih opsi QC Manager.
    • Company: Pilih nama perusahaan tempat QC Manager tersebut bernaung sesuai dengan yang telah didaftarkan sebelumnya.
  • Setelah semua data dipastikan benar, klik tombol Submit. Jika proses berhasil, akan muncul notifikasi "User create successfully".

2. Cara Mengedit Data dan Mengubah Status QC Manager

  • Jika ada perubahan data atau karyawan yang resign, Anda dapat memperbarui profil atau menonaktifkan akun tersebut melalui menu User Management.
  • Gunakan kolom pencarian (Search) di atas tabel untuk mencari username atau nama pengguna yang ingin diubah.
  • Pada kolom Action di baris pengguna tersebut, klik ikon edit yang berwarna kuning (bergambar pensil).
  • Kotak dialog Edit User akan terbuka. Di sini Anda dapat:
    • Memperbaiki kesalahan penulisan nama pada kolom Name.
    • Mengubah status akun dengan memilih opsi inactive pada menu dropdown Status (jika ingin menonaktifkan akun agar tidak bisa digunakan login lagi).
  • Penting: Jika Anda tidak berniat mereset kata sandi pengguna, pastikan kolom New Password dan Confirm New Password dibiarkan kosong sebelum menyimpan.
  • Klik tombol Submit untuk menyimpan perubahan. Jika status berhasil dinonaktifkan, label pada tabel akan berubah menjadi warna merah bertuliskan INACTIVE.

Panduan Menambah QC Officer Baru

Panduan ini menjelaskan langkah-langkah untuk menambahkan dan mengelola akun pengguna dengan peran QC Officer. Berbeda dengan penambahan peran manajerial yang dilakukan oleh admin pusat, proses pembuatan akun ini dilakukan langsung oleh QC Manager untuk anggota tim di perusahaannya masing-masing.

1. Cara Membuat Akun QC Officer Baru

  • Silakan login ke aplikasi QUALIFY menggunakan akun QC Manager Anda.
  • Setelah masuk ke Dashboard, pilih menu User Management pada panel navigasi di sebelah kiri. Halaman ini akan menampilkan daftar QC Officer yang sudah Anda buat sebelumnya.
  • Klik tombol Create User berwarna hijau di bagian pojok kanan atas layar.
  • Pada kotak dialog Create User yang muncul, lengkapi formulir pendaftaran berikut:
    • Name: Masukkan nama pengguna.
    • Username: Masukkan username yang akan digunakan untuk akses login.
    • Email: Masukkan alamat email pengguna.
    • Password & Confirm Password: Buat kata sandi baru (minimal 7 karakter) dan ketik ulang untuk konfirmasi.
    • Role: Pilih opsi QC Officer (sebagai QC Manager, Anda hanya memiliki otorisasi untuk membuat akun dengan peran ini).
  • Klik tombol Submit untuk menyimpan data.

2. Cara Mengedit Data dan Menonaktifkan Akun

  • Pada halaman User Management, gunakan fitur Search (kolom pencarian) di kanan atas tabel untuk mencari nama atau username dari akun yang ingin Anda kelola.
  • Klik ikon edit berwarna kuning (bergambar pensil) pada kolom Action di baris pengguna tersebut.
  • Pada kotak dialog Edit User, Anda dapat merubah nama pengguna di kolom Name atau mengubah Password jika diperlukan.
  • Cara Menonaktifkan Akun: Jika staf tersebut sudah tidak bertugas, Anda dapat mencabut akses login-nya dengan cara mengubah Status pada menu dropdown paling bawah menjadi inactive.
  • Klik tombol Submit untuk menerapkan perubahan. Akun yang berhasil dinonaktifkan akan ditandai dengan label INACTIVE berwarna merah pada tabel dan pengguna tersebut tidak akan bisa login kembali ke dalam sistem.

Panduan Menambah PERPI Officer Baru

Panduan ini menjelaskan langkah-langkah untuk membuat dan mengelola akun pengguna dengan peran PERPI Officer. Sama seperti penambahan QC Manager, hak akses untuk melakukan proses pendaftaran akun ini hanya dimiliki oleh pengguna dengan level PERPI Admin.

1. Cara Membuat Akun PERPI Officer Baru

  • Silakan login ke aplikasi QUALIFY menggunakan akun perpi_admin.
  • Pada panel navigasi di sebelah kiri layar, klik menu User Management.
  • Klik tombol Create User berwarna hijau yang berada di pojok kanan atas halaman.
  • Pada kotak dialog Create User yang muncul, lengkapi formulir pendaftaran:
    • Name: Masukkan nama lengkap pengguna.
    • Username: Masukkan username untuk login.
    • Email: Masukkan alamat email aktif pengguna.
    • Password & Confirm Password: Buat kata sandi untuk pengguna tersebut dan ketik ulang untuk konfirmasi.
    • Role: Pastikan Anda memilih opsi PERPI Officer pada menu dropdown.
    • Company: Berbeda dengan QC Manager yang harus memilih nama perusahaan spesifik, untuk PERPI Officer Anda dapat memilih opsi Total Company (agar memiliki akses tinjauan ke seluruh perusahaan).
  • Klik tombol Submit untuk menyimpan data. Notifikasi "User create successfully" akan muncul jika berhasil.

2. Cara Mengedit Data PERPI Officer

  • Pada menu User Management, gunakan kolom pencarian (Search) untuk mencari nama atau username PERPI Officer yang ingin diedit.
  • Klik ikon edit berwarna kuning (bergambar pensil) pada kolom Action.
  • Di kotak dialog Edit User, Anda dapat merubah Name, Username, atau mereset Password.
  • Penting: Jika Anda hanya ingin mengedit nama/username dan tidak ingin merubah kata sandi, pastikan kolom New Password dan Confirm New Password dalam keadaan kosong (hapus titik-titik kata sandi yang terisi otomatis oleh browser) agar data bisa disimpan.
  • Klik Submit untuk memperbarui data.

3. Cara Menonaktifkan (Inactive) dan Mengaktifkan Kembali (Active) Akun

  • Jika pengurus tersebut sudah tidak bertugas, Anda dapat menonaktifkan akunnya. Pada kotak dialog Edit User, ubah Status pada menu dropdown paling bawah menjadi inactive.
  • Klik Submit. Status pada tabel akan berubah menjadi label merah bertuliskan INACTIVE.
  • Cara Mengaktifkan Kembali: Jika Anda ingin mengaktifkan akun itu lagi, klik ikon edit dan ubah status kembali menjadi active.
  • Catatan Khusus saat Mengaktifkan Ulang: Saat akun diubah menjadi inactive, sistem terkadang mengosongkan kolom Email. Oleh karena itu, sebelum menekan tombol Submit untuk mengaktifkan ulang, pastikan Anda mengetik kembali alamat email pengguna tersebut di kolom yang tersedia agar proses penyimpanan tidak error.
